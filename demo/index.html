V Embeddable Playground

Автоматичная настройка

Вставьте на страницу где вы хотите добавить playground, тег <script> и укажите селектор через атрибут data-selector по которому будет определено какие элементы на странице должны быть превращены в playground. 

<script src="https://unpkg.com/vlang-playground@1" data-selector=".vlang-code">
</script>

Таким образом, элемент: 

<div class="vlang-code">
    println("Hello, Playground!")
</div>

Будет превращен в инстанс playground:

println("Hello, Playground!")

Темная тема

По умолчанию, playground создается со светлой темой, но вы можете задать темную тему прописав data-theme="dark". 

<div class="vlang-code" data-theme="dark">
    println("Hello, Playground!")
</div>
Результат:
println("Hello, Playground!")

Конфигурация

Как вы могли заметить, в предыдущем абзаце мы поменяли тему на темную прописав новый атрибут data-theme.

Похожим образом настраиваются и другие параметры.

Все настройки которые будут показаны далее, также могут быть установлены для тега <script> через который вы подключаете библиотеку. В таком случае они будут применяться для всех инстансов playground.

Запуск кода как тесты

Одной из настроек является выбор конфигурации запуска кода в playground.

На выбор есть две конфигурации:

  1. run default
  2. tests

Чтобы выбрать нужную, используйте атрибут data-configuration.

Например, вы устанавливаете атрибут в значение tests. 

<div class="vlang-code" data-configuration="tests">
fn test_me() {
    assert 100 == 100
}
</div>

Таким образом, вы получите playground в котором кнопка Run будет запускать код как тесты:

Попробуйте нажать кнопку Run, чтобы запустить написанные тесты:

fn test_me() { assert 100 == 100 }

Сокрытие части кода

Playground поддерживают специальные маркеры в коде для указания части кода которая будет показана в playground.

  • //code::start — устанавливает начало фрагмента
  • //code::end — устанавливает конец фрагмента

Так, например, следующий код: 

<div class="vlang-code" data-configuration="tests">
fn test_sum() {
    assert sum(2, 2) == 4
}

//code::start
fn sum(a int, b int) int {
    return a + b
}
//code::end
</div>

Будет конвертирован в playground в котором будет показана только функция sum:

fn test_sum() { assert sum(2, 2) == 4 } //code::start fn sum(a int, b int) int { return a + b } //code::end

Сверху playground появилась новая кнопка +. По ее нажатию разворачивается весь код который был скрыт. Обратите внимание, что скрытый код нельзя редактировать.

Чтобы скрыть эту кнопку используйте настройку:

# data-show-folded-code-button

Устанавливает нужно ли показывать кнопку показа всего кода.

По-умолчанию, true

fn test_sum() { assert sum(2, 2) == 4 } //code::start fn sum(a int, b int) int { return a + b } //code::end

Интерактивные задания с проверкой

Благодаря тому что в playground можно запускать тесты, а также скрывать часть кода, они могут стать прекрасной возможностью для интерактивных заданий.

Вы можете скрыть тесты, показав только функцию где нужно выполнить какое-то задание. Скрыв кнопку показа всего кода, вы уберет возможность заглянуть в тесты, чтобы в них нельзя было подглядеть.

К примеру возьмем следующее простое задание:

Задание:

Напишите тело функции sum, так чтобы все тесты прошли:

fn test_sum() { assert sum(2, 2) == 4 } //code::start fn sum(a int, b int) int { // write your code here } //code::end

Если вы запустите код в таком виде, то вы получите ошибку от компилятора.

Когда вы напишите верный код, то при запуске playground покажет, что все тесты прошли.

Такие небольшие задания могут помочь новичкам быстрее влиться в язык.

Другие настройки

Ниже приведены все доступные настройки, которые были не описаны ранее:

# data-font-size

Устанавливает размер шрифта для заданного playground.

println("Hello, Playground!")

# data-show-line-numbers

Устанавливает необходимо ли показывать номера строк.

По-умолчанию, true

println("Hello, Playground!")

# data-highlight-only

Устанавливает будет ли данный playground readonly. Также в таком playground нельзя запустить код.

По-умолчанию, false

println("Hello, Playground!")

# data-show-footer

Устанавливает показывать ли футер со ссылкой на полноценный playground.

По-умолчанию, true

println("Hello, Playground!")

# data-server

Устанавливает ссылку на сервер на который будут посылаться запросы на запуск из playground.

По-умолчанию, https://play.vlang.foundation

println("Hello, Playground!")

# data-custom-run-button

Устанавливает селектор для кастомной кнопки запуска. Данная настройка может быть использована, если вы хотите встроить playground на страницу и вынести кнопку запуска кода на панель управления.

По-умолчанию, не установлено 

<button class="custom-run-button">Click me</button>

<div class="vlang-code" data-custom-run-button=".custom-run-button">
    println("Hello, Playground!")
</div>

println("Hello, Playground!")
